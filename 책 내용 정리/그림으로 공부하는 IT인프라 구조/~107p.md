### 큐
- 큐의 길이를 확인하는 것으로 레이어의 어디가 막혔는지 알 수 있음  
-> 예) 데이터베이스 성능문제에서  
-> 데이터베이스 큐, OS 런큐, I/O 리퀘스트 큐를 확인하는 것으로 어디가 막혔는지 알 수 있다

- 성능문제에서는 CPU 사용률 뿐만 아니라 런큐의 길이도 확인하는 것이 중요

### 배타제어를 사용할 때
- 장점   
-> 공유 데이터의 정합성을 유지할 수 있음

- 단점  
-> 병렬처리가 불가능

### 배타제어를 사용하지 않을 때 
- 장점   
-> 병렬로 빠른 처리가 가능

- 단점  
-> 동시에 공유데이터를 덮어쓸 경우 부정합이 발생 할 수 있음
